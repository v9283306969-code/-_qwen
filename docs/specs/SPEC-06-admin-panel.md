## 2. Бизнес-контекст

### Назначение

Интерфейс для владельцев бизнеса и менеджеров. Позволяет управлять товарами, отслеживать заказы, просматривать пользователей, корректировать ранги/бонусы в экстренных случаях, настраивать параметры системы (проценты, периоды).

### Роли доступа (RBAC)

| Роль | Доступ |
|---|---|
| **Super-Admin** | Полный доступ, включая настройки бонусов и правки транзакций |
| **Manager** | Заказы, Пользователи, Товары. Без правки финансов и настроек. |
| **Analyst** | Только чтение (Аналитика, Отчёты). |

## 3. Функциональные разделы (Страницы)

### 3.1 Dashboard
- KPI: Продажи за день, Активные пользователи, Сумма бонусов к выплате.
- Графики: Динамика регистраций по промокодам.

### 3.2 Управление пользователями (связь с SPEC-03)
- Таблица всех пользователей (поиск по email, телефону, коду).
- Просмотр дерева сети (визуализация кто-кого привёл).
- Действия:
  - Просмотр баланса.
  - **Ручное изменение ранга** (если баг в авто-пересчёте).
  - **Блокировка пользователя** (за фрод).
  - **Списание/Начисление бонусов** (ручная корректировка).

### 3.3 Управление товарами (связь с SPEC-01)
- Дублирование функционала Strapi или прямой CRUD через сервис.
- **⚠️ Strapi — Express-заглушка (scaffold).** До Этапа 4 Strapi не работает. CRUD товаров через сервис (SPEC-01), затем интеграция с настоящим Strapi.
- Быстрое изменение цен для акций.
- Управление видимостью (is_active).

### 3.4 Управление заказами (связь с SPEC-02)
- Список заказов с фильтрами.
- Ручная смена статуса (для техподдержки).
- **Принудительный возврат** -> Запуск механизма отката бонусов (SPEC-03 FR-8).

### 3.5 Настройки системы (SPEC-03 rank_settings)
- Изменение базовых процентов (bonus_1_level_base, и т.д.).
- Изменение множителей рангов.
- Изменение периода задержки выплат (hold_days).

## 4. Требования к API (Требования к Бэкенду)

Для работы админки сервисы должны предоставить следующие методы (если их нет):

| Сервис | Требуемый метод | Описание |
|---|---|---|
| SPEC-03 | `POST /internal/users/:id/block` | Блокировка пользователя |
| SPEC-03 | `POST /internal/users/:id/adjust-rank` | Ручная смена ранга |
| SPEC-03 | `POST /internal/bonuses/manual-adjust` | Ручная корректировка баланса |
| SPEC-02 | `POST /internal/orders/:id/force-return` | Принудительный возврат с откатом бонусов |
| SPEC-03 | `GET /internal/admin/network-tree?user_id=...` | Получение структуры сети для визуализации |

---

## 5. Безопасность

- Доступ только по отдельному домену (admin.mlecp.ru) или пути `/admin`.
- Strict RBAC на уровне API (Admin Guard).
- Логирование **всех** действий администратора (кто изменил ранг, кто сменил процент).
