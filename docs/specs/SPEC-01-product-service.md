## 2. Бизнес-контекст

### Назначение сервиса

Управление каталогом товаров: хранение информации о товарах, категориях, ценах, наличии. Обеспечивает чтение для витрины сайта и запись для админки.

### Пользователи сервиса

| Кто | Как использует |
|---|---|
| Покупатель | Просмотр каталога, фильтрация, поиск, карточка товара |
| Администратор | CRUD товаров, управление категориями, ценами, остатками |
| Order Service | Запрашивает наличие и цену товара при оформлении заказа |

### Связь с бизнес-моделью

Две категории товаров: косметика и БАДы. Раздельная витрина (разные лендинги), но единый учёт. Интеграция со Strapi CMS для описаний и медиа. (`docs/business-models/multi-level-ecommerce.md`)

## 3. Архитектурные решения

### Тип сервиса

Микросервис с REST API для внутренних вызовов + GraphQL-резолверы для агрегации данных.

### Принятые решения

| Решение | Почему | Альтернативы (отклонены) |
|---|---|---|
| Кэш товаров в Redis | Каталог — самая читаемая часть. Redis снижает нагрузку на PostgreSQL в 10+ раз. | Без кэша → PostgreSQL bottleneck при росте трафика |
| Интеграция со Strapi через вебхуки | Strapi хранит контент (описания, фото), а Product Service — бизнес-данные (цены, остатки). Разделение ответственности. | Хранить всё в одной БД → Strapi теряет смысл как Headless CMS |
| Пагинация cursor-based | Лучше для бесконечной прокрутки на мобильных, быстрее offset | Offset-пагинация → медленно при больших смещениях |

### Зависимости

| Зависимость | Тип | Назначение |
|---|---|---|
| PostgreSQL | Инфраструктура | Основная БД (товары, категории, цены) |
| Redis | Инфраструктура | Кэш ответов API |
| Strapi CMS | Внешний Service | Хранение описаний товаров, медиа-файлов |
| SPEC-02 (Order Service) | Микросервис | Запрос наличия/цен при оформлении заказа |

## 4. Модель данных

### Схема БД

```sql
-- Категории
categories (id, name, slug, description, parent_id, image_url, is_active, sort_order, created_at, updated_at)

-- Товары
products (id, name, slug, category_id, description, is_active, created_at, updated_at)

-- Цены товаров (история изменений цен)
product_prices (id, product_id, price, old_price, currency, valid_from, valid_to)

-- Остатки (по складам)
inventory (id, product_id, warehouse_id, quantity, reserved, updated_at)

-- Медиа (связь со Strapi)
product_media (id, product_id, strapi_media_id, image_url, alt_text, is_primary, sort_order)
```

### Таблицы

#### categories

| Поле | Тип | Nullable | Описание | Пример |
|---|---|---|---|---|
| id | UUID | No | Уникальный идентификатор | 550e8400-e29b-... |
| name | VARCHAR(100) | No | Название категории | "Уходовая косметика" |
| slug | VARCHAR(100) | No | URL-идентификатор | "care-cosmetics" |
| description | TEXT | Yes | Описание категории | "Кремы, лосьоны, сыворотки..." |
| parent_id | UUID | Yes | ID родительской категории (дерево) | null или UUID |
| image_url | VARCHAR(500) | Yes | URL изображения категории | "/img/cats/care.jpg" |
| is_active | BOOLEAN | No | Показывать на витрине | true |
| sort_order | INT | No | Порядок сортировки | 10 |
| created_at | TIMESTAMP | No | Время создания | 2026-05-25 10:00:00 |
| updated_at | TIMESTAMP | No | Время обновления | 2026-05-25 12:00:00 |

#### products

| Поле | Тип | Nullable | Описание | Пример |
|---|---|---|---|---|
| id | UUID | No | Уникальный идентификатор | 550e8400-e29b-... |
| name | VARCHAR(200) | No | Название товара | "Крем увлажняющий X" |
| slug | VARCHAR(200) | No | URL-идентификатор | "cream-moisturizing-x" |
| category_id | UUID | No | ID категории | UUID категории |
| description | TEXT | Yes | Краткое описание | "Увлажняющий крем для лица..." |
| is_active | BOOLEAN | No | Показывать на витрине | true |
| created_at | TIMESTAMP | No | Время создания | 2026-05-25 10:00:00 |
| updated_at | TIMESTAMP | No | Время обновления | 2026-05-25 12:00:00 |

#### product_prices

| Поле | Тип | Nullable | Описание | Пример |
|---|---|---|---|---|
| id | UUID | No | Уникальный идентификатор | ... |

| valid_from | TIMESTAMP | No | Начало действия цены | 2026-05-25 00:00:00 |
| valid_to | TIMESTAMP | Yes | Конец действия цены | null |

#### inventory

| Поле | Тип | Nullable | Описание | Пример |
|---|---|---|---|---|
| id | UUID | No | Уникальный идентификатор | ... |
| product_id | UUID | No | ID товара | UUID |
| warehouse_id | UUID | No | ID склада | UUID |
| quantity | INT | No | Количество на складе | 150 |

### Связи с другими таблицами

| Таблица | Тип связи | Поле | Описание |
|---|---|---|---|
| products -> categories | N:1 | products.category_id -> categories.id | Товар принадлежит одной категории |
| categories -> categories | N:1 | categories.parent_id -> categories.id | Дерево категорий (подкатегории) |
| product_prices -> products | N:1 | product_prices.product_id -> products.id | История цен для одного товара |
| inventory -> products | N:1 | inventory.product_id -> products.id | Остатки для одного товара (по складам) |

## 5. API контракты

### Входящие запросы (GraphQL-резолверы / внутренние REST)

| Endpoint | Метод | Описание | Вход | Выход |
|---|---|---|---|---|
| `/internal/products` | GET | Список товаров (пагинация, фильтры) | `?category=...&q=...&cursor=...&limit=20` | `{ products, hasMore, nextCursor }` |
| `/internal/products/:id` | GET | Детали товара | `id` в пути | `{ product, price, stock, media }` |
| `/internal/categories` | GET | Дерево категорий | — | `{ categories }` |
| `/internal/products/health` | GET | Health check | — | `{ status: "ok" }` |

### Исходящие запросы (к другим сервисам)

| Сервис | Endpoint | Когда вызывается | Зачем |
|---|---|---|---|
| Strapi CMS | `GET /api/products` | При синхронизации контента | Получить описания и медиа-файлы |

### Ошибки

| Код | HTTP | Описание | Когда возвращается |
|---|---|---|---|
| PRODUCT_NOT_FOUND | 404 | Товар не найден или не активен | Запрос несуществующего `id` |
| CATEGORY_NOT_FOUND | 404 | Категория не найдена | Запрос несуществующей категории |
| INVALID_CURSOR | 400 | Некорректный курсор пагинации | Повреждённый cursor token |

## 7. Нефункциональные требования

| Требование | Значение | Как проверяется |
|---|---|---|
| Производительность (чтение) | TTFB < 100ms (p95) для cached-запросов | Нагрузочный тест (k6/Artillery) |
| Производительность (чтение без кэша) | TTFB < 300ms (p95) | Нагрузочный тест |
| Надёжность | Сервис восстанавливается при рестарте Docker | docker-compose restart → health check |
| Безопасность | Только внутренние сервисы вызывают API (без прямого доступа из интернета) | Network policy в Docker |
| Масштабируемость | Горизонтальное масштабирование (多个 инстансов за load balancer) | Запуск 2+ контейнеров через Docker Compose |

## 8. Тесты

### Unit-тесты

| Сценарий | Вход | Ожидаемый результат |
|---|---|---|
| Создание товара | Валидные данные | Товар создан в БД, возвращён с UUID |
| Обновление цены | Новый ценa | Запись в product_prices, старая цена сохраняется (valid_to) |
| Фильтрация по категории | category_id | Возвращаются только товары этой категории |
| Пагинация | cursor=..., limit=10 | Возвращаются следующие 10 товаров, hasNext=true |

### Интеграционные тесты

| Сценарий | Что мокать | Что проверять |
|---|---|---|
| Создание товара → получение | — | Созданный товар возвращается GET-запросом |
| Кэш Redis → инвалидация | Redis | При обновлении товара кэш инвалидируется |
| Health check | — | Возвращает { status: "ok", db: "connected" } |
