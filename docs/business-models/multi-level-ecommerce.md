## 1. Общее описание идеи

### Концепция

Построение **интернет-магазина на headless CMS** с микросервисной архитектурой, ориентированного на реализацию косметики и БАДов через партнёрскую сеть. Ключевая особенность — **многоуровневая мотивационная модель**, где каждый участник цепочки получает вознаграждение с продажи по своему промокоду.

### Каталог товаров

- **Категория 1:** Косметика (уходовые средства, декоративная косметика, профессиональные средства)
- **Категория 2:** БАДы (витамины, добавки, спортивное питание, косметические комплексы)
- **Общий учёт:** Единая база данных для товарного учёта
- **Раздельная витрина:** Разные лендинги/каталоги для каждой категории

### Ключевая идея

Клиент может покупать напрямую через сайт, но если он использует промокод партнёра — срабатывает цепочка вознаграждений. Это создаёт **виральный рост**: каждый участник мотивирован привлекать новых людей, потому что получает процент с их продаж.

### Мобильная стратегия

**Сайт (responsive + PWA)** — Progressive Web App. Нативное приложение отложено до 10 000+ активных пользователей. Причина: каждое дополнительное звено в цепочке регистрации (ссылка → App Store → скачать → зарегистрироваться) убивает конверсию на 40-60%. MLM-модель требует минимального трения.

## 3. Структура участников

### Иерархия уровней

```
                    ┌──────────────────┐
                    │  Клиент 1 ур.    │  ← Торговый агент (Лидер сети)
                    │  (Промокод ААА)  │     Бонус: 8-15% со ВСЕХ продаж сети
                    └──────┬───────────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
        ┌─────┴──┐  ┌──────┴──┐  ┌──────┴──┐
        │Клиент 2│  │Клиент 2 │  │Клиент 2 │  ← Реализатор
        │уровня  │  │уровня   │  │уровня   │     Бонус: 10-20% с личных продаж
        │(BBB)   │  │(CCC)    │  │(DDD)    │
        └────┬───┘  └────┬────┘  └────┬────┘
             │           │            │
        ┌────┴───┐  ┌────┴────┐  ┌────┴────┐
        │Клиент 3│  │Клиент 3 │  │Клиент 3 │  ← Конечный покупатель
        │уровня  │  │уровня   │  │уровня   │     Скидка: 10-15%
        │(покуп.)│  │(покуп.) │  │(покуп.) │
        └────────┘  └─────────┘  └─────────┘
```

### Связь уровней через промокод

| Связь | Как работает |
|---|---|
| 1 → 2 | Агент передаёт свой промокод реализатору. Реализатор регистрируется по этому коду → привязка к сети агента |
| 2 → 3 | Реализатор передаёт тот же промокод покупателю. Покупатель регистрируется/покупает по коду |
| 3 → Магазин | Покупатель покупает товар на сайте, промокод автоматически привязан к заказу → цепочка сработала |

**Ключевой принцип:** Один промокод = одна цепочка. Все продажи по этому промокоду = бонусы для участников цепочки.

## 4. Система рангов

### Клиент 1 уровня (Торговый агент)

Критерий: размер сети + оборот сети (последние 3 месяца, пересчёт 1-го числа каждого месяца)

| Ранг | Активных 2 ур. | Оборот сети | Множитель | Бонус 1 ур. | Бонус 2 ур. |
|---|---|---|---|---|---|
| Старт | 0 | 0 | 1.0× | 8% | 10% |
| Бронза | 3 | 50 продаж | 1.2× | 10% | 12% |
| Серебро | 10 | 500 продаж | 1.5× | 12% | 15% |
| Золото | 30 | 2000 продаж | 2.0× | 15% | 20% |

**Правила:**
- Понижение ранга: если за скользящее окно 3 месяцев не выполнено условие
- Пересчёт: 1-го числа каждого месяца автоматически
- Уведомление: email/SMS за 7 дней до возможного понижения

---

### Клиент 2 уровня (Реализатор)

Критерий: личные продажи конечным покупателям (последние 3 месяца)

| Ранг | Продаж 3 ур. | Личный оборот | Множитель | Бонус 2 ур. |
|---|---|---|---|---|
| Старт | 0 | 0 | 1.0× | 10% |
| Бронза | 10 | — | 1.2× | 12% |
| Серебро | 50 | 100 000₽ | 1.5× | 15% |
| Золото | 200 | 500 000₽ | 2.0× | 20% |

**Правила:**
- Считаются только продажи непосредственно клиентам 3 уровня
- Личный оборот — сумма продаж за период
- Понижение ранга аналогично 1 уровню

---

### Клиент 3 уровня (Конечный покупатель)

Критерий: количество покупок за период (персональная лояльность, без сетевой активности)

| Ранг | Покупок за период | Скидка |
|---|---|---|
| Покупатель | 1 | 10% |
| Постоянный | 5 за 3 мес. | 12% |
| VIP | 15 за 6 мес. | 15% |

**Правила:**
- Скидка применяется автоматически через промокод
- Ранг не влияет на бонусы сети — только на персональную скидку
- Повышение автоматическое, понижение — если не было покупок за период

## 6. Механика промокодов

### Что такое промокод в системе

Промокод — это уникальный идентификатор, привязанный к аккаунту клиента 1 уровня. Он связывает всю цепочку продаж и отслеживает бонусы.

### Жизненный цикл промокода

```
1. Агент регистрируется → получает уникальный промокод (QR-код)
2. Агент передаёт промокод реализатору
3. Реализатор регистрируется по промокоду → привязка к сети
4. Реализатор передаёт тот же промокод покупателю
5. Покупатель покупает на сайте → промокод привязан к заказу
6. Система начисляет бонусы по цепочке
```

### Правила промокодов

| Правило | Описание |
|---|---|
| Один промокод = один агент | Промокод уникален, не может быть у двух агентов |
| Регистрироваться можно только по промокоду | Без промокода — только как покупатель без сети |
| Промокод можно применять многократно | Каждый покупатель, который использует код, привязывается к сети |
| QR-код = промокод | Для печати на визитках, упаковках, буклетах |
| Неактивные промокоды | Если агент 3 месяца не имеет активности — промокод деактивируется |

### Пример применения

| Покупатель | Промокод | Заказ | Бонусы |
|---|---|---|---|
| Иван (3 ур.) | AGENT-123 | 3000₽ | Агент (1 ур.): 240₽ (8%), Реализатор (2 ур.): 300₽ (10%), Иван: скидка 300₽ |

## 7. Механика возвратов

### Общая логика

Косметика и БАДы по закону РФ — **невозвратные товары** после передачи покупателю. Единственное исключение — брак, обнаруженный в момент передачи.

### Процесс возврата

| Этап | Что происходит |
|---|---|
| 1. Покупатель обнаружил брак | При получении — можно отказаться. Товар не принимается |
| 2. Возврат в системе | Заказ помечается как «возврат». Бонусы не начисляются |
| 3. Если бонусы уже начислены | Все транзакции откатываются в обратном порядке: 2 ур → 1 ур → магазин |
| 4. Задержка выплаты | Бонусы выплачиваются с задержкой 7-14 дней, чтобы исключить возвраты |

### Откат транзакций (пример)

Исходная продажа: 2000₽, бонусы начислены:
- 1 ур.: 160₽
- 2 ур.: 200₽

Возврат через 3 дня:
- 200₽ снимается со счёта 2 ур.
- 160₽ снимается со счёта 1 ур.
- 2000₽ возвращается покупателю (или не списывается, если оплата при получении)

## 8. Юридическая модель

### Форма сотрудничества

| Форма | Для кого | Налог | Лимит | Плюсы |
|---|---|---|---|---|
| Самозанятый (НПД) | Клиенты 1 и 2 ур. | 4-6% | 2.4 млн ₽/год | Простая регистрация через приложение «Мой налог» |
| ИП (УСН 6%) | Крупные агенты | 6% | 200 млн ₽/год | Выше лимит, официальные выплаты |
| Физлицо (13% НДФЛ) | Разовые продажи | 13% | Нет | Без регистрации, но сложно для выплат |

### Рекомендация

- **Основная форма:** самозанятость (90% случаев)
- **Для крупных агентов:** ИП (при превышении лимита)
- **Разовые выплаты физлицам:** только для тестирования до регистрации

### Оферта

| Раздел оферты | Что регулирует |
|---|---|
| Условия начисления бонусов | Формула, сроки, порядок |
| Возвраты и откаты | Механика, задержка выплаты |
| Ранговая система | Условия перехода и понижения |
| Ответственность сторон | Кто за что отвечает |
| Конфиденциальность | Защита данных |
| Прекращение сотрудничества | Как выйти, что происходит с бонусами |

### Юридический статус модели

MLM (multilevel marketing) **легален в РФ** при наличии:
- Реального товара (не финансовая пирамида)
- Прозрачных условий
- Договорных отношений

**Важно:** проконсультироваться с юристом, специализирующимся на MLM/сетевом маркетинге.

## 11. Сильные стороны

| Преимущество | Почему это важно |
|---|---|
| Виральный рост | Каждый участник = бесплатный продавец. Сеть растёт без затрат на рекламу |
| Низкий порог входа | Стать агентом — только регистрация. Не нужен склад, опыт, стартовый капитал |
| Гибкость ценообразования | Настраиваемые бонусы через админку → адаптация к рынку |
| Headless CMS + микросервисы | Масштабируемость, лёгкие обновления, разделение ответственности |
| Прозрачная ранговая система | Понятные правила → доверие → больше активность |
| Задержка выплат + откат | Снижение риска возвратов без потери мотивации |
| Два направления товаров | Косметика + БАДы = расширяющаяся аудитория |
| PWA вместо приложения | Минимальное трение для регистрации и покупок, работает офлайн |
