## Когда применять

Перед шагом **3.4.1 (Frontend — Next.js)** или любым этапом, когда начинается реализация визуального UI. Особенно важно когда дизайн НЕ был обсуждён на этапе архитектуры.

## Проблема

Без дизайн-гайда фронтенд создается «как получится» — с дефолтными цветами, закруглениями, тенями. Переделка потом стоит 30-50% времени фронтенд-разработки. Но полный дизайн в Figma не нужен — достаточно задокументированной палитры, типографики и принципов.

## Процедура

### Шаг 1: Получить референс от пользователя

- Попросить ссылку на сайт, MHTML/PDF файл, или скриншоты
- **НЕ пытаться «прочитать» цвета из большого файла** — MHTML 25MB нельзя распарсить через llm
- Спросить прямые вопросы: фон (какой цвет?), заголовки (какой цвет?), описания (какой цвет?)
- Если пользователь прислал файл — попросить уточнить: «какой фон — светлый или тёмный?», «какой цвет текста описаний?»

### Шаг 2: Верификация цветов с пользователем

После первой попытки — спросить: «соответствует ли палитра референсу?». Это критично — ошибка в цвете фона меняет ВЕСЬ бренд (как было при путанице «золотой фон» vs «слоновая кость»).

**Не переходить к шагу 3 ПОКА пользователь не подтвердил палитру.**

### Шаг 4: Конвертация цветов в HSL для shadcn/ui

shadcn/ui использует HSL (hue saturation lightness). Конвертация:

```
Hex #B8933B → HSL 42 52% 48%
Hex #FDFCF3 → HSL 54 50% 97%
Hex #000000 → HSL 0 0% 10% (не 0% — шрифт чуть мягче)
```

Использовать онлайн-конвертер или Tailwind color picker.

### Шаг 5: Учесть особые требования

| Требование | Как применить |
|---|---|
| **Без теней** | Добавить `* { box-shadow: none !important; }` в globals.css |
| **Без закруглений** | `--radius: 0` + `* { border-radius: 0 !important; }` |
| **Премиальный минимализм** | Воздух > декор, Playfair Display weight 400 (не 700!) |
| **Золото на слоновой кости** | Золото — ЕДИНСТВЕННЫЙ цветной элемент страницы |

### Шаг 6: shadcn/ui globals.css — готовый шаблон

```css
@import "tailwindcss";

@theme inline {
  --font-sans: "Inter", sans-serif;
  --font-serif: "Playfair Display", serif;

  --color-background: <HEX>;
  --color-foreground: <HEX>;
  --color-primary: <HEX>;
  --color-primary-foreground: <HEX>;
  --color-card: <HEX>;
  --color-card-foreground: <HEX>;
  --color-secondary: <HEX>;
  --color-border: <HEX>;
  --color-ring: <HEX>;
  --radius: 0;
}

* { border-radius: 0 !important; }
```

## Типичные ошибки

| Ошибка | Почему | Как избежать |
|---|---|---|
| Неправильно определённый фон | MHTML 25MB → попытка читать все inline-image CSS → неверный вывод | **Спросить пользователя напрямую** — «фон светлый или тёмный?» |
| Неправильный оттенок акцента | Золото `#D4A853` vs `#B8933B` — выглядит по-разному | Верифицировать с пользователем ДО продолжения |
| Тёмная тема по умолчанию для светлого бренда | «Для ювелирки тёмная тема лучше» — но референс светлый | **Референс всегда прав.** Не предполагать — следовать. |
| Шрифт weight 700 вместо 400 | Жирный = маркетинг, тонкий = editorial/premium | Для премиум-бренда Playfair Display 400, не 700 |

## Критическое правило

**Верификация с пользователем — это не лишняя трата токенов.** Один вопрос «соответствует ли палитра референсу?» экономит 20+ часов переделки фронтенда.
